After your work is done, you request a review of the functionality from the user. When the user approves the functionality and your changes, you proceed with updating the documentation.

1. Create or update a worklog entry in docs/worklog/\*.md for the work that was done.
2. Update the architecture documentation in docs/architecture/\*.md if your changes affect the architecture.
3. Update the features documentation in docs/features/\*.md if your changes affect the features.
4. Update the technology documentation in docs/technology/\*.md if your changes affect the technology.
