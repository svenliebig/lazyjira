# Worklog

Chronological record of development sessions. Each entry documents what was built, why decisions were made, and any non-obvious special cases worth remembering.

## Structure

```
worklog/
  README.md                          ← this file
  YYYY-MM-DD_<slug>.md               ← one file per session
```

## Entry format

Each session file uses the following sections:

| Section | Purpose |
|---------|---------|
| **Goal** | What the session set out to accomplish |
| **Completed work** | Bulleted list of what was actually done |
| **Bug fixes** | Root causes and solutions for bugs found and fixed |
| **Special cases** | Non-obvious implementation choices worth remembering |
| **Open items** | Anything explicitly left unfinished |

Sessions are named by the date they occurred and a short slug describing the primary work.
